# Agent Lab 🧪

This is your learning lab for building small AI tools for your marketing agency.

Think of this folder as a **workshop**. Each tool you build gets its own little
workbench (a folder inside `projects/`), so experiments never get tangled up
with each other.

You don't need to memorize any of this. When you want to do something, just ask
in plain English and your assistant will handle the technical parts.

---

## What's in here

| Folder / File   | What it is, in plain words                                              |
|-----------------|-------------------------------------------------------------------------|
| `projects/`     | One folder per tool you build. Your actual work lives here.             |
| `notes/`        | Your learning notes. Includes a plain-language glossary of terms.       |
| `.env.example`  | A template showing where secret keys (like your AI password) will go.   |
| `.gitignore`    | A safety list of files the computer should never share or back up.      |
| `README.md`     | This file — the front-desk guide to the whole lab.                      |

---

## How you'll work here

1. You describe a tool you want ("I want something that rewrites headlines").
2. Your assistant creates a new folder for it inside `projects/`.
3. You test it, learn from it, and keep it — or throw it away. No harm done.

Each project is self-contained, so you can never "break the lab" by experimenting.

---

## The one thing to remember about safety

To use AI in your tools, you'll need an **API key** — think of it as a password
that lets your tools talk to the AI. Treat it like a credit-card number:

- It goes in a private file called `.env` (never shared, never posted online).
- The `.gitignore` file in this lab is already set up to keep it private for you.

When you're ready to get one, just ask your assistant to walk you through it.
